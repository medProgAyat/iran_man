## بخش صفر: پیش‌نیازهای ذهنی (این قسمت را نخوانی هم مشکلی نیست ولی اگر بخوانی، همه چیز را بهتر می‌فهمی)

همهٔ برنامه‌هایی که می‌بینید، از یک ماشین‌حساب ساده گرفته تا یک بازی پیچیده مثل «اتاق فرار» یا شبکه اجتماعی، از تعدادی «دستور» (Code) تشکیل شده‌اند که پشت سر هم اجرا می‌شوند. برنامه‌نویس مثل یک آشپز است که به یک آشپزخانه (کامپیوتر) می‌گوید: «اول پیاز را خرد کن، بعد توی روغن بریز، بعد نمک بزن...». دستورها را به زبان‌هایی می‌نویسیم که کامپیوتر بفهمد؛ یکی از شیرین‌ترین این زبان‌ها «پایتون» است. پایتون آن‌قدر به حرف زدن آدم‌ها نزدیک است که خیلی‌ها به آن «زبان انگلیسی برنامه‌نویسی» می‌گویند. مثلاً برای چاپ یک جمله روی صفحه کافی است بنویسی:

```python
print("سلام دنیا!")
```

همین. خیلی ساده، نه؟

حالا تصور کن می‌خواهیم یک بازی بسازیم. بازی یعنی چه؟ یعنی تصاویر روی صفحه حرکت کنند، صدا پخش بشود، بازیکن با کیبورد یا موس یا حتی صدایش کارهایی بکند. برای این‌کار ما نیاز به یک «موتور بازی‌سازی» داریم؛ یعنی یک جعبه ابزار آماده که کارهای تکراری مثل کشیدن عکس روی صفحه، پخش صدا و گرفتن دکمه‌های کیبورد را برایمان انجام بدهد. Pygame دقیقاً همین است: یک کتابخانهٔ رایگان و قدرتمند برای ساخت بازی‌های دوبُعدی با پایتون.

حالا از آن جذاب‌تر: می‌خواهیم به جای کیبورد، با حرف زدن بازی را کنترل کنیم و بازی هم به ما جواب بدهد. این یعنی دو جادوی دیگر: «تشخیص گفتار» (Speech-to-Text) که صدای فارسی ما را به متن تبدیل کند و «تبدیل متن به گفتار» (Text-to-Speech) که متن فارسی را با صدای طبیعی بخواند.

قرار است تمام این‌ها را از صفر یاد بگیری. نگران هیچ چیز نباش. هر خط کد را با دلیل توضیح می‌دهم و در نهایت یک بازی کامل خواهیم داشت که:

1. یک مربع قهرمان دارد که باید آیتم‌های درخشان (ستاره‌ها) را جمع کند.
2. دشمن‌هایی دارد که اگر بهشان بخوری، جانت کم می‌شود.
3. کنترل مربع فقط با صدای فارسی تو انجام می‌شود: «برو بالا»، «برو پایین»، «برو چپ»، «برو راست».
4. هر وقت آیتم جمع کنی، بازی با صدای فارسی بگوید: «آفرین! یک ستاره جمع کردی».
5. هر وقت ببازی، بازی بگوید: «باختی! امتیاز نهایی تو ... است».

نکته مهم: من همهٔ مفاهیم را طوری توضیح می‌دهم که بتوانی آن را حفظ کنی. پس هر جا لازم باشد از مثال‌های روزمره، داستان‌سرایی و تکرار استفاده می‌کنم. با من همراه باش.

## بخش یک: نصب ابزارها (اولین قرار ما با پایتون و دوستانش)

اگر هیچ چیز بلد نیستی، باید اول پایتون را روی کامپیوترت نصب کنی. مثل این است که می‌خواهی یک ماشین برانی؛ اول باید موتور ماشین (پایتون) را داشته باشی، بعد لوازم یدکی (کتابخانه‌ها) را اضافه کنی.

### قدم ۱: نصب پایتون

1. مرورگرت را باز کن و برو به وب‌سایت رسمی پایتون: `python.org`
2. روی دکمهٔ بزرگ زرد رنگ که نوشته «Download Python» کلیک کن. (نسخه‌ای که می‌بینی احتمالاً ۳.۱۱ یا بالاتر است. فرقی نمی‌کند، فقط عدد اولش ۳ باشد).
3. فایل دانلودی را اجرا کن.
   - **ویندوز**: حتماً تیک گزینهٔ «Add Python to PATH» را بزن. (این کار خیلی مهم است، مثل این است که کلید ماشین را بگیری دستت. اگر این تیک را نزنی، بعداً دستورات پایتون را در خط فرمان نمی‌شناسد.)
   - سپس روی Install Now کلیک کن و صبر کن تا تمام شود.
4. برای تست، خط فرمان (Command Prompt در ویندوز یا Terminal در مک و لینوکس) را باز کن. در ویندوز دکمهٔ Windows کیبورد را بزن، عبارت `cmd` را تایپ کن و Enter بزن.
5. در پنجرهٔ سیاه، تایپ کن: `python --version` و Enter بزن. اگر چیزی شبیه `Python 3.11.4` دیدی، یعنی موفق شدی. تبریک! تو الان یک برنامه‌نویس پایتون هستی، حتی اگر هنوز یک خط کد ننوشته باشی.

## بخش دو: مبانی پایتون در ۲۰ دقیقه (حتی اگر الفبا هم بلد نباشی!)

همان‌طور که قول دادم، هیچ پیش‌نیازی نمی‌خواهد. زبان برنامه‌نویسی یعنی مجموعه‌ای از کلمات که به کامپیوتر می‌گوید چکار کند. پایتون مثل یک دستیار باهوش است که جمله‌های ما را می‌خواند و انجام می‌دهد. اول چند مفهوم اصلی را به ساده‌ترین شکل ممکن مرور می‌کنیم. می‌توانی این بخش را مثل یک داستان بخوانی.

### ۱. متغیرها: جعبه‌های نام‌گذاری شده

فرض کن یک جعبه داری و می‌خواهی داخل آن عدد ۵ را بگذاری. روی جعبه با ماژیک می‌نویسی «x». حالا هر وقت گفتی «x»، در واقع به همان جعبه اشاره می‌کنی.

```python
x = 5
y = 10
z = x + y   # حالا z می‌شود ۱۵
```

می‌توانی داخل جعبه هر چیزی بگذاری: عدد، متن (که به آن رشته یا String می‌گویند)، لیست...

```python
name = "امیر"
age = 25
```

در پایتون نیازی نیست بگویی جعبه از چه جنسی است؛ خودش می‌فهمد. به این می‌گویند «نوع‌دهی پویا».

### ۲. شرط‌ها: اگر این طور شد، آن کار را بکن

زندگی پر از انتخاب است. در کدنویسی هم با `if` و `else` تصمیم می‌گیریم.

```python
score = 100
if score >= 50:
    print("قبول شدی")
else:
    print("رد شدی")
```

اینجا کامپیوتر چک می‌کند: آیا score بزرگتر یا مساوی ۵۰ است؟ اگر بله، خط اول را اجرا کن. در غیر این صورت، خط بعد از `else` را اجرا کن. دقت کن که بعد از `if` و `else` حتماً دو نقطه می‌گذاریم و خط بعدی را با کمی فاصله (تورفتگی یا Indent) می‌نویسیم. این تورفتگی در پایتون اجباری است و جان کد است! معمولاً ۴ فاصله یا یک دکمهٔ Tab.

### ۳. حلقه‌ها: کارهای تکراری را خسته نشو

اگر بخواهی ده بار بگویی «سلام»، لازم نیست ده خط بنویسی. از حلقهٔ `for` استفاده می‌کنی:

```python
for i in range(10):
    print("سلام")
```

`range(10)` یعنی اعداد صفر تا ۹ را به ترتیب به متغیر `i` بده. اما چون ما از `i` استفاده نکردیم، فقط دستور چاپ ده بار تکرار می‌شود.

حلقهٔ `while` هم هست: تا وقتی یک شرط برقرار است، تکرار کن.

```python
counter = 0
while counter < 5:
    print(counter)
    counter = counter + 1
```

این برنامه از ۰ تا ۴ را چاپ می‌کند. مواظب باش که شرط هیچ‌وقت تمام نشود وگرنه برنامه قفل می‌کند. (همیشه یک راه خروج بگذار.)

### ۴. توابع: کارخانه‌های کوچک

گاهی یک کار تکراری داری و می‌خواهی اسمی رویش بگذاری تا هر وقت صدا زدی، انجام شود. تابع یعنی همین: یک بسته کد با نام مشخص.

```python
def say_hello(name):
    print("سلام " + name)

say_hello("سارا")   # خروجی: سلام سارا
say_hello("علی")    # خروجی: سلام علی
```

`def` یعنی تعریف تابع. داخل پرانتز می‌توانی ورودی بگیری. هر وقت تابع را صدا بزنی، کدهای داخلش اجرا می‌شود. اینطوری کدت تمیز و خوش‌ساخت می‌شود.

### ۵. لیست‌ها و دیکشنری‌ها (جعبه‌های پیشرفته)

لیست یعنی چند چیز را پشت سر هم توی یک ردیف بگذاری:

```python
fruits = ["سیب", "موز", "پرتقال"]
print(fruits[0])   # سیب (چون شمارش از صفر شروع می‌شود)
```

دیکشنری یعنی برای هر چیزی یک برچسب اسمی بزنی:

```python
student = {"name": "احمد", "age": 22, "grade": "A"}
print(student["name"])   # احمد
```

تا اینجا کافی است. همین‌ها پایه‌های اصلی هر برنامه‌ای هستند. حالا برویم سراغ اصل مطلب.

## بخش سه: دنیای Pygame – از هیچ، یک بازی بساز

پنجره‌ای تصور کن که قرار است نقاشی‌های ما توی آن جان بگیرند. Pygame این پنجره را می‌سازد و مدیریت می‌کند. ساختار هر بازی Pygame از چهار مرحلهٔ طلایی تشکیل شده است:

1. **مقدمات**: کتابخانه را وارد کن، پنجره بساز، اسم بگذار.
2. **حلقهٔ اصلی بازی (Game Loop)**: یک حلقهٔ بی‌نهایت که تا وقتی بازیکن بازی را نبسته، ادامه دارد.
3. **مدیریت رویدادها**: در هر لحظه ببین بازیکن دکمه‌ای زده، موس را تکان داده، یا پنجره را بسته است؟
4. **به‌روزرسانی و رسم**: اشیا را حرکت بده، برخوردها را چک کن، همه چیز را روی صفحه بکش.

بیا این چهار قدم را با یک مثال ساده ببینیم. یک فایل جدید در VS Code درست کن و اسمش را بگذار `game.py`. کد زیر را خط‌به‌خط توضیح می‌دهم:

```python
import pygame
import sys

# 1. مقدمات
pygame.init()   # روشن کردن موتور Pygame
width, height = 800, 600   # ابعاد پنجره
screen = pygame.display.set_mode((width, height))   # ساختن پنجره
pygame.display.set_caption("بازی اول من")   # عنوان پنجره

# رنگ‌ها را تعریف می‌کنیم (RGB)
WHITE = (255, 255, 255)
RED   = (255, 0, 0)

# مختصات اولیهٔ یک مربع قرمز
x, y = 400, 300
speed = 5

clock = pygame.time.Clock()  # برای کنترل سرعت بازی

# 2. حلقهٔ اصلی
while True:
    # 3. مدیریت رویدادها
    for event in pygame.event.get():
        if event.type == pygame.QUIT:   # اگر دکمهٔ بستن پنجره زده شد
            pygame.quit()
            sys.exit()
    
    # دریافت کلیدهای فشرده شده
    keys = pygame.key.get_pressed()
    if keys[pygame.K_UP]:
        y -= speed
    if keys[pygame.K_DOWN]:
        y += speed
    if keys[pygame.K_LEFT]:
        x -= speed
    if keys[pygame.K_RIGHT]:
        x += speed
    
    # 4. رسم
    screen.fill(WHITE)   # صفحه را با رنگ سفید پاک کن
    pygame.draw.rect(screen, RED, (x, y, 50, 50))   # مربع قرمز در مختصات (x,y) با ابعاد ۵۰x۵۰
    
    pygame.display.flip()   # صفحه را بروز کن (انگار نقاشی را نشان بده)
    clock.tick(60)   # سرعت ۶۰ فریم در ثانیه
```

### تشریح خط‌به‌خط (اینجا جای حفظ کردن است)

- `import pygame`: کتابخانه Pygame را وارد می‌کند تا بتوانیم از امکاناتش استفاده کنیم.
- `import sys`: برای بستن کامل برنامه با `sys.exit()`.
- `pygame.init()`: تمام بخش‌های Pygame را راه‌اندازی می‌کند. مثل زدن استارت ماشین.
- `screen = pygame.display.set_mode((800,600))`: یک پنجره با عرض ۸۰۰ و ارتفاع ۶۰۰ پیکسل می‌سازد. پیکسل یعنی نقطه‌های ریز صفحه نمایش.
- `WHITE = (255, 255, 255)`: رنگ سفید در فضای RGB. عدد اول قرمزی، دوم سبزی، سوم آبی. هرکدام بین ۰ تا ۲۵۵. ترکیب ۲۵۵ از همه یعنی سفید. ۰ از همه یعنی سیاه.
- `while True:`: حلقه‌ای که تا ابد تکرار می‌شود (تا وقتی با `break` یا بستن برنامه متوقفش کنیم). این قلب بازی است.
- `for event in pygame.event.get():`: تمام اتفاقات مثل فشردن دکمه، حرکت موس و ... در یک صف جمع شده‌اند. اینجا یکی‌یکی آن‌ها را می‌خوانیم.
- `if event.type == pygame.QUIT:`: اگر رویداد از نوع خروج (بستن پنجره) بود، بازی را تمام کن.
- `keys = pygame.key.get_pressed()`: یک لیست از وضعیت تمام دکمه‌های کیبورد می‌گیرد. هر دکمه یا ۰ است (فشرده نشده) یا ۱ (فشرده شده).
- `if keys[pygame.K_LEFT]: x -= speed`: اگر دکمهٔ جهت چپ پایین نگه داشته شده، مختصات x پنج تا کم شود. (حرکت به چپ یعنی کاهش x. چون در Pygame نقطهٔ (۰,۰) بالا-چپ است و با افزایش x به راست می‌رویم و افزایش y به پایین.)
- `screen.fill(WHITE)`: کل صفحه را با رنگ سفید نقاشی می‌کند. این کار باعث می‌شود تصویر قبلی پاک شود. بدون این خط، مربع‌ها روی هم کشیده می‌شوند و یک خط ممتد درست می‌کنند.
- `pygame.draw.rect(screen, RED, (x, y, 50, 50))`: یک مستطیل (مربع) قرمز روی صفحه می‌کشد. مختصات گوشهٔ بالا-چپ آن `(x, y)` و عرض و ارتفاعش ۵۰ پیکسل.
- `pygame.display.flip()`: آنچه روی صفحهٔ پنهان کشیده‌ایم را به نمایش واقعی می‌آورد. مثل ورق زدن کتاب.
- `clock.tick(60)`: به بازی می‌گوید حداکثر ۶۰ فریم در ثانیه اجرا شود. اگر کامپیوتر قوی‌تر بود، بیشتر از ۶۰ نمی‌رود. اینطوری سرعت بازی روی همهٔ سیستم‌ها یکسان می‌ماند.

کد را run کن (در VS Code دکمهٔ مثلثی بالا سمت راست، یا در ترمینال بنویس `python game.py`). باید یک پنجره با مربع قرمز ببینی که با جهت‌نماها حرکت می‌کند. این اولین بازی توست! به همین راحتی.

### مفهوم اسپرایت (Sprite) و شیء‌گرایی

برای بازی‌های پیچیده‌تر، مدیریت تک‌تک مربع‌ها و دشمن‌ها با متغیرهای جدا سخت می‌شود. اینجا پای «شیءگرایی» به میان می‌آید. یک کلاس (Class) مثل یک قالب کلوچه است. قالب، شکل کلوچه را تعیین می‌کند و با آن می‌توانی بی‌نهایت کلوچه با ویژگی‌های مشابه درست کنی. در Pygame کلاس `Sprite` یک قالب آماده برای موجودیت‌های بازی است.

بگذار کلاس قهرمان را بسازیم:

```python
class Player(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.Surface((50, 50))
        self.image.fill(RED)
        self.rect = self.image.get_rect()
        self.rect.center = (400, 300)
        self.speed = 5
    
    def update(self, keys):
        if keys[pygame.K_LEFT]:
            self.rect.x -= self.speed
        if keys[pygame.K_RIGHT]:
            self.rect.x += self.speed
        if keys[pygame.K_UP]:
            self.rect.y -= self.speed
        if keys[pygame.K_DOWN]:
            self.rect.y += self.speed
```

توضیح: `Player` از `pygame.sprite.Sprite` ارث‌بری می‌کند، یعنی تمام ویژگی‌های یک اسپرایت را دارد. `__init__` متد سازنده است؛ وقتی یک بازیکن جدید می‌سازی اجرا می‌شود. `self.image` ظاهر اسپرایت است (اینجا یک مربع ۵۰×۵۰ قرمز). `self.rect` مستطیل محاطی اسپرایت است که برای موقعیت و برخوردها استفاده می‌شود. متد `update` کار حرکت را انجام می‌دهد.

حالا در حلقهٔ اصلی، کافی است:

```python
player = Player()
# در حلقه:
keys = pygame.key.get_pressed()
player.update(keys)
screen.fill(WHITE)
screen.blit(player.image, player.rect)   # رسم بازیکن
```

`blit` یعنی تصویر را روی صفحه در موقعیت مشخص‌شده بچسبان.

بیا با همین روش دشمن‌ها و آیتم‌ها را بسازیم. اما قبل از آن، باید کمی دربارهٔ برخورد (Collision) بدانیم.

### برخوردها: وقتی قهرمان به دشمن می‌خورد

Pygame تابع‌های خیلی خوبی برای تشخیص برخورد دارد. `pygame.sprite.spritecollide()` چک می‌کند که آیا یک اسپرایت با گروهی از اسپرایت‌ها برخورد کرده است یا نه. مثلاً:

```python
if pygame.sprite.spritecollide(player, enemy_group, False):
    # برخورد رخ داده
    print("آخ!")
```

پارامتر سوم اگر True باشد، دشمن حذف می‌شود (مثلاً برای جمع‌آوری آیتم‌ها). ما در بازی‌مان از این استفاده می‌کنیم.

## بخش چهار: تشخیص گفتار فارسی (از صدای تو تا متن)

حالا می‌رسیم به جادوی واقعی. می‌خواهیم برنامه حرف‌های فارسی ما را بفهمد و به متن تبدیل کند. کتابخانه `SpeechRecognition` این کار را با کمک سرویس‌های آنلاین (مثل گوگل) انجام می‌دهد. مراحل کلی:

1. از میکروفون صدا ضبط کن.
2. فایل صوتی را به سرویس آنلاین بفرست.
3. متن تشخیص‌داده‌شده را دریافت کن.

توجه: نیاز به اینترنت داری چون از موتور گوگل استفاده می‌کند. (بعداً نسخهٔ آفلاین را هم یاد می‌گیری.)

### اولین تست تشخیص گفتار

یک فایل جدید به اسم `stt_test.py` بساز:

```python
import speech_recognition as sr

# یک شیء شناساگر بساز
r = sr.Recognizer()

# از میکروفون پیش‌فرض استفاده کن
with sr.Microphone() as source:
    print("صحبت کن...")
    # تنظیم برای نویز محیط (یک ثانیه گوش می‌دهد تا نویز پس‌زمینه را بسنجد)
    r.adjust_for_ambient_noise(source)
    # گوش دادن به صدای کاربر تا وقتی سکوت کند
    audio = r.listen(source)

# حالا صدای ضبط شده را با گوگل به متن تبدیل کن (فارسی)
try:
    text = r.recognize_google(audio, language="fa-IR")
    print("تو گفتی: " + text)
except sr.UnknownValueError:
    print("متوجه نشدم")
except sr.RequestError as e:
    print("خطا در اتصال به سرویس گوگل: {0}".format(e))
```

کد را اجرا کن. بعد از پیام «صحبت کن...» یک جملهٔ فارسی بگو، مثلاً «سلام دنیا». کمی صبر کن، اگر اینترنت وصل باشد، متن را چاپ می‌کند. چند نکتهٔ حیاتی:

- `adjust_for_ambient_noise(source)`: میکروفون یک ثانیه به صدای محیط گوش می‌دهد تا سطح نویز را تشخیص دهد و بعد گفتار را بهتر متوجه شود. همیشه این خط را بگذار، مخصوصاً اگر محیط شلوغ است.
- `language="fa-IR"`: زبان فارسی ایران را مشخص می‌کند. حواست باشد اگر لهجه داری، بیشتر کلمات را درست تشخیص می‌دهد ولی نه همه را. هرچند برای دستورات ساده مثل «بالا» عالی عمل می‌کند.
- اگر بلندگو و میکروفون روی هدفون است، سیستم آن را به عنوان ورودی انتخاب می‌کند. می‌توانی میکروفون پیش‌فرض را عوض کنی.

### چالش: برنامه تا وقتی تو حرف نزنی، قفل می‌شود

متد `listen(source)` تا وقتی سکوت نکنی منتظر می‌ماند و حلقهٔ بازی را قفل می‌کند. در بازی نمی‌توانیم اینطور رفتار کنیم. راه حل: استفاده از Thread (نخ) یعنی یک کارگر جداگانه که فقط کار گوش دادن را انجام می‌دهد و مزاحم کارگر اصلی (بازی) نمی‌شود. این تکنیک را در بخش ترکیب کامل یاد می‌دهیم.

## بخش پنج: تبدیل متن به گفتار فارسی (حرف زدن بازی با تو)

حالا نوبت آن است که بازی جوابت را بدهد. با gTTS می‌توانیم یک رشتهٔ فارسی را به یک فایل mp3 تبدیل کنیم و بعد با Pygame پخش کنیم. مراحل:

1. متن فارسی را بگیر.
2. با gTTS یک شیء صوتی بساز.
3. آن را در یک فایل موقت (مثلاً `output.mp3`) ذخیره کن.
4. با Pygame یا کتابخانه‌ای دیگر پخش کن.

چون Pygame خودش می‌تواند mp3 پخش کند، نیازی به کتابخانهٔ اضافی نیست.

### نصب gTTS

اگر قبلاً دستور `pip install gtts` را زدی، که هیچ. اگر نه، همین الان بزن.

کد نمونه:

```python
from gtts import gTTS
import pygame
import os

def speak_persian(text):
    # ساختن صدای فارسی
    tts = gTTS(text=text, lang='fa', slow=False)
    filename = "temp_voice.mp3"
    tts.save(filename)
    
    # پخش با Pygame
    pygame.mixer.init()
    pygame.mixer.music.load(filename)
    pygame.mixer.music.play()
    while pygame.mixer.music.get_busy():  # صبر کن تا تمام شود
        pygame.time.Clock().tick(10)
    pygame.mixer.music.unload()   # آزاد کردن فایل
    os.remove(filename)   # پاک کردن فایل موقت

# امتحان
speak_persian("سلام بازیکن عزیز! به بازی من خوش آمدی.")
```

توضیح: `lang='fa'` یعنی فارسی. `slow=False` یعنی سرعت عادی. `save` فایل mp3 را می‌سازد. `pygame.mixer.music.load` فایل را بارگذاری می‌کند و `play` پخش می‌کند. حلقهٔ `while` صبر می‌کند تا پخش تمام شود، بعد فایل پاک می‌شود. این کار باعث می‌شود هربار فایل جدید جایگزین شود.

توجه: ممکن است هنگام پخش، بازی کمی مکث کند. در بازی نهایی می‌توانیم پخش صدا را هم در یک نخ جداگانه ببریم تا بازی یخ نزند. اما برای سادگی فعلاً همین کافی است.

## بخش شش: ترکیب همه چیز – بازی صوتی نهایی

حالا وقت آن است که همهٔ این قطعات را مثل یک پازل کنار هم بچینیم. یادت باشد: ما قول یک بازی را دادیم که با صدا کنترل شود و خودش هم حرف بزند. بیا طراحی کلی را مرور کنیم:

- یک پنجرهٔ ۸۰۰×۶۰۰.
- یک بازیکن (مربع آبی) که با کلیدهای کیبورد هم حرکت می‌کند (برای تست) اما فرمان اصلی از صدا می‌آید.
- دشمن‌ها (مربع‌های قرمز) که از بالای صفحه به پایین می‌ریزند.
- ستاره‌های طلایی که بازیکن باید جمع کند.
- برخورد با دشمن = باختن.
- جمع‌آوری ستاره = افزایش امتیاز + پیغام صوتی.
- در گوشهٔ صفحه، متن فارسی وضعیت مثل «امتیاز: ۵» نمایش داده می‌شود.
- نخ جداگانه برای گوش دادن به میکروفون و فرمان‌های فارسی.

### فایل‌ها و ساختار

یک فایل اصلی `main_game.py` می‌سازیم. برای زیبایی و مدیریت بهتر، کلاس‌ها را در یک فایل جدا مثلاً `sprites.py` بگذاریم ولی می‌توانیم همه چیز را در یک فایل بگذاریم تا ساده‌تر باشد. من همه را در `main_game.py` می‌آورم و توضیح می‌دهم.

### کد نهایی با توضیح خط‌به‌خط (برای حفظ کردن)

این کد بلند است اما نترس. قدم‌قدم می‌رویم جلو.

```python
import pygame
import sys
import random
import speech_recognition as sr
from gtts import gTTS
import threading
import os
import time

# -------------------------------
#  بخش اول: تنظیمات اولیه Pygame
# -------------------------------
pygame.init()
pygame.mixer.init()  # برای پخش صدا
WIDTH, HEIGHT = 800, 600
screen = pygame.display.set_mode((WIDTH, HEIGHT))
pygame.display.set_caption("بازی صوتی فارسی - علی کوچولو")

# رنگ‌ها (RGB)
WHITE = (255, 255, 255)
BLACK = (0, 0, 0)
RED = (255, 0, 0)
BLUE = (0, 0, 255)
GOLD = (255, 215, 0)

# فونت برای نمایش متن فارسی (فونت پیش‌فرض سیستم. در ویندوز معمولاً فونت Tahoma دارد)
font = pygame.font.Font(None, 36)  # None یعنی فونت پیش‌فرض، اندازه ۳۶
clock = pygame.time.Clock()

# -------------------------------
#  بخش دوم: کلاس‌های بازی
# -------------------------------
class Player(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.Surface((50, 50))
        self.image.fill(BLUE)
        self.rect = self.image.get_rect()
        self.rect.center = (WIDTH//2, HEIGHT-100)
        self.speed = 8
        self.score = 0
    
    def update(self, direction=None):
        # اگر direction مشخص باشد (از فرمان صوتی) همان را اعمال کن
        # اگر None باشد، از کیبورد بخوان (برای تست آسان)
        if direction:
            if direction == "بالا":
                self.rect.y -= self.speed
            elif direction == "پایین":
                self.rect.y += self.speed
            elif direction == "چپ":
                self.rect.x -= self.speed
            elif direction == "راست":
                self.rect.x += self.speed
        else:
            keys = pygame.key.get_pressed()
            if keys[pygame.K_LEFT] or keys[pygame.K_a]:
                self.rect.x -= self.speed
            if keys[pygame.K_RIGHT] or keys[pygame.K_d]:
                self.rect.x += self.speed
            if keys[pygame.K_UP] or keys[pygame.K_w]:
                self.rect.y -= self.speed
            if keys[pygame.K_DOWN] or keys[pygame.K_s]:
                self.rect.y += self.speed
        
        # محدود کردن بازیکن به داخل پنجره
        if self.rect.left < 0:
            self.rect.left = 0
        if self.rect.right > WIDTH:
            self.rect.right = WIDTH
        if self.rect.top < 0:
            self.rect.top = 0
        if self.rect.bottom > HEIGHT:
            self.rect.bottom = HEIGHT

class Enemy(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.Surface((40, 40))
        self.image.fill(RED)
        self.rect = self.image.get_rect()
        self.rect.x = random.randint(0, WIDTH-40)
        self.rect.y = random.randint(-100, -40)   # بالای صفحه شروع می‌شود
        self.speed_y = random.randint(3, 8)
    
    def update(self):
        self.rect.y += self.speed_y
        if self.rect.top > HEIGHT:  # اگر از پایین خارج شد، حذف شود
            self.kill()

class Star(pygame.sprite.Sprite):
    def __init__(self):
        super().__init__()
        self.image = pygame.Surface((30, 30))
        self.image.fill(GOLD)
        self.rect = self.image.get_rect()
        self.rect.x = random.randint(0, WIDTH-30)
        self.rect.y = random.randint(-100, -30)
        self.speed_y = random.randint(2, 5)
    
    def update(self):
        self.rect.y += self.speed_y
        if self.rect.top > HEIGHT:
            self.kill()

# -------------------------------
#  بخش سوم: تابع تبدیل متن به گفتار (در نخ جدا)
# -------------------------------
def speak(text):
    def run():
        try:
            tts = gTTS(text=text, lang='fa', slow=False)
            filename = "temp_speech.mp3"
            tts.save(filename)
            # باید mixer را حتماً init کرده باشیم.
            # اگر mixer مشغول پخش قبلی است، صبر می‌کنیم
            while pygame.mixer.music.get_busy():
                pygame.time.Clock().tick(10)
            pygame.mixer.music.load(filename)
            pygame.mixer.music.play()
            while pygame.mixer.music.get_busy():
                pygame.time.Clock().tick(10)
            pygame.mixer.music.unload()
            os.remove(filename)
        except Exception as e:
            print("خطای پخش صدا:", e)
    # یک نخ جداگانه می‌سازیم تا بازی قفل نشود
    t = threading.Thread(target=run)
    t.start()

# -------------------------------
#  بخش چهارم: متغیرهای اصلی بازی
# -------------------------------
player = Player()
all_sprites = pygame.sprite.Group()
enemies = pygame.sprite.Group()
stars = pygame.sprite.Group()

all_sprites.add(player)

# متغیرهای زمان‌بندی برای ایجاد دشمن و ستاره
ENEMY_EVENT = pygame.USEREVENT + 1
pygame.time.set_timer(ENEMY_EVENT, 1000)  # هر ۱ ثانیه یک دشمن

STAR_EVENT = pygame.USEREVENT + 2
pygame.time.set_timer(STAR_EVENT, 2000)   # هر ۲ ثانیه یک ستاره

# -------------------------------
#  بخش پنجم: سیستم تشخیص گفتار در نخ جداگانه
# -------------------------------
# یک صف (Queue) برای انتقال فرمان از نخ به حلقهٔ اصلی
import queue
command_queue = queue.Queue()

def listen_for_command():
    recognizer = sr.Recognizer()
    with sr.Microphone() as source:
        recognizer.adjust_for_ambient_noise(source, duration=1)
        print("سیستم صوتی آماده است. فرمان بدهید...")
    while True:
        try:
            with sr.Microphone() as source:
                # هر بار صبر نمی‌کنیم adjust کند تا سریع‌تر باشد، مگر در شروع.
                audio = recognizer.listen(source, timeout=1, phrase_time_limit=2)
            text = recognizer.recognize_google(audio, language="fa-IR")
            print("شنیدم:", text)
            # کلمات کلیدی را چک کن
            if "بالا" in text:
                command_queue.put("بالا")
            elif "پایین" in text:
                command_queue.put("پایین")
            elif "چپ" in text:
                command_queue.put("چپ")
            elif "راست" in text:
                command_queue.put("راست")
            elif "امتیاز" in text:
                # فقط برای تست، می‌توانی بعداً استفاده کنی
                pass
        except sr.UnknownValueError:
            pass
        except sr.RequestError as e:
            print("خطای سرویس:", e)
        except Exception as e:
            print("خطا:", e)

# راه‌اندازی نخ شنود
listener_thread = threading.Thread(target=listen_for_command, daemon=True)
listener_thread.start()

# -------------------------------
#  بخش ششم: حلقهٔ اصلی بازی
# -------------------------------
game_over = False
while not game_over:
    # ۱. مدیریت رویدادها
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            game_over = True
        elif event.type == ENEMY_EVENT:
            enemy = Enemy()
            all_sprites.add(enemy)
            enemies.add(enemy)
        elif event.type == STAR_EVENT:
            star = Star()
            all_sprites.add(star)
            stars.add(star)
    
    # ۲. دریافت فرمان صوتی از صف (بدون مسدود کردن)
    voice_direction = None
    try:
        voice_direction = command_queue.get_nowait()
    except queue.Empty:
        pass
    
    # ۳. به‌روزرسانی بازیکن با فرمان صوتی یا کیبورد (اولویت با صدا)
    if voice_direction:
        player.update(direction=voice_direction)
    else:
        player.update()   # از کیبورد بخواند
    
    # ۴. به‌روزرسانی دشمن‌ها و ستاره‌ها
    enemies.update()
    stars.update()
    
    # ۵. بررسی برخورد بازیکن با ستاره‌ها (جمع‌آوری)
    collected_stars = pygame.sprite.spritecollide(player, stars, True)
    for star in collected_stars:
        player.score += 10
        speak("آفرین! یک ستاره جمع کردی. امتیازت شد " + str(player.score))
    
    # ۶. بررسی برخورد با دشمن‌ها (باختن)
    if pygame.sprite.spritecollide(player, enemies, False):
        speak("باختی! امتیاز نهایی تو " + str(player.score) + " است.")
        game_over = True
        # می‌توانیم حلقه را بشکنیم، اما ممکن است صدا هنوز تمام نشده باشد. ما مستقیم می‌رویم.
        break
    
    # ۷. رسم
    screen.fill(BLACK)  # پس‌زمینه مشکی
    all_sprites.draw(screen)
    
    # نمایش امتیاز روی صفحه
    score_text = font.render("امتیاز: " + str(player.score), True, WHITE)
    screen.blit(score_text, (10, 10))
    
    # راهنمایی
    hint_text = font.render("بگو: بالا، پایین، چپ، راست", True, WHITE)
    screen.blit(hint_text, (WIDTH//2 - 150, 10))
    
    pygame.display.flip()
    clock.tick(60)

# پایان بازی: صبر می‌کنیم تا صدای باختن پخش شود بعد خارج می‌شویم.
pygame.time.wait(3000)  # ۳ ثانیه صبر کن
pygame.quit()
sys.exit()
```

### شرح کامل و نکات حفظ کردنی

**۱. وارد کردن کتابخانه‌ها و پیکربندی:**

- `pygame.init()` و `pygame.mixer.init()`: اولی کل Pygame را راه می‌اندازد، دومی سیستم صدا را جداگانه فعال می‌کند تا بتوانیم mp3 پخش کنیم.
- فونت `None, 36` از فونت پیش‌فرض سیستم استفاده می‌کند. اگر متن فارسی نمایش داده نشد، می‌توانی یک فونت فارسی (مثل `arial.ttf`) دانلود کنی و جایگزین کنی.

**۲. کلاس Player:**

- در `update(direction=None)` اگر direction داده شود (از صدا) مستقیماً حرکت می‌کند، در غیر این‌صورت کلیدهای کیبورد را می‌خواند. این طراحی باعث می‌شود هم بشود با صدا بازی کرد، هم برای تست با کیبورد.
- محدودیت حرکت با `if self.rect.left < 0: ...` مانع خروج بازیکن از صفحه می‌شود. همیشه این محدودیت را بگذار.

**۳. کلاس‌های Enemy و Star:**

- `random.randint` برای شروع از بالای صفحه با مختصات تصادفی.
- `self.kill()` در متد `update` باعث می‌شود اگر از صفحه خارج شدند، از گروه حذف شوند و منابع آزاد شود. خیلی مهم است.

**۴. تابع speak(text):**

- این تابع یک نخ جدید می‌سازد و gTTS را در آن اجرا می‌کند. چرا؟ چون ساخت و پخش فایل mp3 کمی زمان می‌برد و اگر در حلقهٔ اصلی باشد، بازی یخ می‌زند و فریم‌ها می‌افتد. با `threading` این کار در پس‌زمینه انجام می‌شود.
- قبل از پخش، منتظر می‌مانیم اگر صدای قبلی هنوز تمام نشده، تمام شود. `get_busy()` وضعیت پخش را نشان می‌دهد.
- فایل موقت `temp_speech.mp3` بعد از پخش پاک می‌شود.

**۵. سیستم رویدادهای زمان‌بندی شده:**

- `pygame.time.set_timer(ENEMY_EVENT, 1000)`: هر ۱۰۰۰ میلی‌ثانیه (۱ ثانیه) یک رویداد سفارشی به صف رویدادها اضافه می‌شود. در حلقه با `if event.type == ENEMY_EVENT` آن را می‌گیریم و یک دشمن جدید می‌سازیم. این خیلی بهتر از استفاده از شمارنده‌های دستی است.

**۶. نخ تشخیص گفتار:**

- `queue.Queue()` یک صف امن برای تبادل داده بین نخ‌ها است. نخ شنود هر وقت فرمانی تشخیص داد، آن را در صف می‌گذارد (`put`).
- در حلقهٔ اصلی با `command_queue.get_nowait()` تلاش می‌کنیم یک فرمان بگیریم بدون این که منتظر بمانیم. اگر صف خالی بود، خطای `Empty` رخ می‌دهد که با `except queue.Empty` از آن می‌گذریم.
- `recognizer.listen(source, timeout=1, phrase_time_limit=2)`: هر بار تا ۲ ثانیه گوش می‌دهد و اگر ۱ ثانیه سکوت شد، قطع می‌کند. این تنظیم باعث می‌شود بتوانیم پشت سر هم فرمان بدهیم.
- تشخیص کلمات کلیدی: ما کل عبارت را می‌گیریم و اگر شامل «بالا» بود، فرمان «بالا» را در صف می‌گذاریم. توجه: ممکن است بگوید «برو بالا»، پس بهتر است بررسی شامل بودن (`in`) انجام دهیم نه تساوی دقیق.
- `daemon=True` در Thread باعث می‌شود با بسته شدن برنامهٔ اصلی، نخ شنود هم خودبه‌خود بسته شود.

**۷. حلقهٔ اصلی:**

- اگر فرمان صوتی وجود داشت، بازیکن را با آن آپدیت می‌کنیم. اولویت با صدا است. اگر خواستی می‌توانی ترکیبی عمل کنی.
- `pygame.sprite.spritecollide(player, stars, True)`: برخورد بازیکن با گروه ستاره‌ها را چک می‌کند، ستاره‌های برخورد کرده حذف می‌شوند (True). خروجی لیست ستاره‌های جمع‌آوری شده است. برای هر کدام امتیاز اضافه و صدای تشویق پخش می‌شود.
- برخورد با دشمن: اگر برخوردی بود، پیام باخت پخش می‌شود، حلقه می‌شکند.
- `pygame.time.wait(3000)` سه ثانیه صبر می‌کند تا صدای باختن کامل پخش شود، بعد پنجره بسته می‌شود. می‌توانی به جای آن یک حلقهٔ کوچک بگذاری و رویدادها را پردازش کنی، اما این ساده‌ترین راه است.

### رفع ایرادهای احتمالی

**فونت فارسی نمایش داده نمی‌شود:**

- در ویندوز می‌توانی از فونت Tahoma استفاده کنی. کد زیر را جایگزین خط فونت کن:

  ```python
  font = pygame.font.SysFont("tahoma", 36)
  ```

  در لینوکس ممکن است نیاز به نصب فونت فارسی داشته باشی.

**خطای میکروفون:**

- اگر PyAudio نصب نشد، می‌توانی از کتابخانه‌های دیگر مثل `sounddevice` استفاده کنی، اما ساده‌ترین راه نصب PyAudio از wheel است. برای رفع مشکل در ویندوز، از سایت `https://www.lfd.uci.edu/~gohlke/pythonlibs/#pyaudio` فایل مناسب پایتون خودت را دانلود و با `pip install` نصب کن.

**صدای gTTS قطعه قطعه یا قطع می‌شود:**

- اینترنت ضعیف یا تحریم باعث می‌شود. می‌توانی از کتابخانه‌ی `edge-tts` استفاده کنی که از سرویس مایکروسافت استفاده می‌کند و کیفیت خوبی برای فارسی دارد. ولی برای سادگی همان gTTS خوب است.

**تاخیر در تشخیص گفتار:**

- هر بار `adjust_for_ambient_noise` را اجرا نکن؛ من فقط اول یک بار گذاشتم. در نسخهٔ بالا در حلقهٔ while دوباره `with sr.Microphone()` باز می‌کنیم. این کار `adjust` را اجرا نمی‌کند (چون داخل همان with نیست). دقت کن که در کد نخ، یک `with sr.Microphone() as source:` بیرون حلقه گذاشتم تا تنظیم نویز اولیه انجام شود. ولی داخل حلقه دوباره `with sr.Microphone() as source:` دارم. این ممکن است هر بار میکروفون را باز و بسته کند، کمی کندی ایجاد می‌کند. بهتر است یک بار میکروفون را باز کنیم و همان شیء source را استفاده کنیم. بیا کد نخ را بهینه کنیم:

```python
def listen_for_command():
    recognizer = sr.Recognizer()
    mic = sr.Microphone()
    with mic as source:
        recognizer.adjust_for_ambient_noise(source, duration=1)
    while True:
        try:
            with mic as source:
                audio = recognizer.listen(source, timeout=1, phrase_time_limit=2)
            text = recognizer.recognize_google(audio, language="fa-IR")
            # بقیه کد...
        except:
            pass
```

این نسخه بهتر است. ما در فایل نهایی از همین استفاده می‌کنیم.

## بخش هفت: آفلاین کردن تشخیص گفتار با Vosk (اختیاری ولی حرفه‌ای)

تا اینجا وابسته به اینترنت بودیم. اگر بخواهی در اوج حرفه‌ای بودن، بازی‌ات بدون اینترنت هم کار کند، باید از Vosk استفاده کنی. Vosk یک موتور تشخیص گفتار آفلاین است که مدل فارسی هم دارد. نصب آن کمی سنگین است ولی ارزشش را دارد.

### قدم‌ها

1. نصب Vosk:

   ```bash
   pip install vosk
   ```

2. نصب کتابخانه‌ی صدا (PyAudio یا sounddevice). ما PyAudio را داریم.
3. دانلود مدل فارسی از سایت Vosk: `https://alphacephei.com/vosk/models` و فایل `vosk-model-fa-0.5` را بگیر. (حدود ۱.۵ گیگابایت). فایل را از زیپ درآور و کنار پروژه بگذار.
4. کد نمونه:

```python
import vosk
import pyaudio
import json

model = vosk.Model("vosk-model-fa-0.5")  # مسیر پوشه مدل
recognizer = vosk.KaldiRecognizer(model, 16000)

mic = pyaudio.PyAudio()
stream = mic.open(format=pyaudio.paInt16, channels=1, rate=16000, input=True, frames_per_buffer=4000)
stream.start_stream()

print("آفلاین: صحبت کن...")
while True:
    data = stream.read(4000, exception_on_overflow=False)
    if recognizer.AcceptWaveform(data):
        result = json.loads(recognizer.Result())
        text = result.get("text", "")
        print("گفتی:", text)
        # اینجا می‌توانی همان تحلیل فرمان را انجام دهی
```

برای ترکیب با Pygame، این کد را در یک نخ می‌گذاری و فرمان‌ها را به صف می‌فرستی. دقت کن که Vosk متن کامل جمله را برمی‌گرداند (مانند gTTS). کیفیت مدل فارسی Vosk خوب است و برای دستورات کوتاه عالی جواب می‌دهد. اگر فضای کافی داری، شدیداً پیشنهاد می‌کنم.

## بخش نه: دیباگ و خطاهای رایج (نجات از باتلاق)

در این سفر، حتماً به خطاهایی برمی‌خوری. نترس! این‌ها دوست تو هستند.

**۱. `ModuleNotFoundError: No module named 'pygame'`**

- یعنی Pygame نصب نیست. دستور `pip install pygame` را در ترمینال اجرا کن. مطمئن شو که در همان محیط پایتونی هستی که VS Code استفاده می‌کند.

**۲. خطای microphone: `OSError: No Default Input Device Available`**

- میکروفون به درستی تشخیص داده نشده. در ویندوز بررسی کن میکروفون فعال باشد. گاهی تنظیمات حریم خصوصی میکروفون را مسدود کرده.

**۳. `pygame.error: mixer not initialized`**

- قبل از پخش صدا، باید `pygame.mixer.init()` را صدا بزنی. من در اول بازی این کار را کردم.

**۴. متن فارسی در پنجره Pygame به صورت مربع یا علامت سوال نمایش داده می‌شود.**

- فونت پیش‌فرض از کاراکترهای فارسی پشتیبانی نمی‌کند. فونت فارسی دلخواه (مثل `BNazanin.ttf`) را در کنار فایل بگذار و اینطور استفاده کن:

  ```python
  font = pygame.font.Font("BNazanin.ttf", 24)
  ```

**۵. بازی کند است یا صدای gTTS خیلی طول می‌کشد.**

- هر بار ساخت MP3 و پخش آن حدود ۱-۲ ثانیه زمان می‌برد. برای کاهش تاخیر، می‌توانی به جای gTTS از `pyttsx3` استفاده کنی که آفلاین و سریع است، اما صدای فارسی خوبی ندارد. یا از فایل‌های از پیش ساخته شده استفاده کن. یا یک صف از فایل‌های صوتی آماده داشته باش.

**۶. خطای `UnboundLocalError` یا ارجاع به متغیر تعریف نشده.**

- یعنی متغیری که می‌خواهی بخوانی هنوز مقداری نگرفته. مثلاً `player.score` وقتی `player` هنوز ساخته نشده. ترتیب کدها را چک کن.

**۷. حلقهٔ بازی زود تمام می‌شود و پنجره بسته می‌شود.**

- احتمالاً `pygame.quit()` بی‌موقع صدا زده شده، یا شرط `game_over` اشتباه است. کدت را خط‌به‌خط با نسخهٔ نهایی مقایسه کن.

**۸. تشخیص گفتار فارسی کلمات را اشتباه می‌شنود.**

- محیط خلوت باشد، میکروفون نزدیک دهان، واضح صحبت کن. می‌توانی از `recognize_google(audio, language="fa-IR", show_all=True)` برای دیدن احتمالات جایگزین استفاده کنی.
